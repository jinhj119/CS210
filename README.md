# Summarize the project and what problem it was solving.
This C++ project is a grocery item tracking application. It reads grocery purchase data from an input file, counts each item's appearance, and allows users to search for item frequencies, displays all item counts, and displays a histogram of all item counts. It solves the problem of manually counting how many times items appear in a purchase record by automatically processing data and presenting it in an organized format.

# What did you do particularly well?
I separated the menu system, file processing, and data management into functions within a class. This made the code easier to read, maintain, and debug. I also used a map to store and retrieve item frequencies, making the data easier to search and display.

# Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
I think I could add features like sorting the items alphabetically or by frequency. Another improvement would be allowing users to update grocery data through the program instead of through the input file.

# Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
The most challenging part of the project was the file-reading and the use of maps. I read C++ documentation to figure out how to implement file-reading and maps. 

# What skills from this project will be particularly transferable to other projects or course work?
This project helped my understanding of object-oriented programming, file input/output, and the use of containers. 

# How did you make this program maintainable, readable, and adaptable?
I used a class-based structure with functions that each perform a specific task. I separated declarations into a header file and implementations into a source file.
